Тапните по любой ссылке или кнопке в мобильном WebKit-браузере, и по умолчанию
он нарисует поверх элемента полупрозрачный прямоугольник на всё время, пока палец
прижат. Цвет берётся из платформы, а не от вас, обычно это синий или серый
прямоугольник, который игнорирует ваш border-radius и обрезается по боксу
элемента. Это явный признак, что интерфейс — веб-страница, а не приложение,
и срабатывает это на каждом вашем элементе управления: ссылки в
навигации, карточки, кнопки-иконки, все подряд.

Исправление — одно наследуемое объявление, но при его применении нужно ещё
избежать двух ошибок.

## Почему это важно

На этом сайте плавающая кнопка «На этой странице» (та, что открывает оглавление
на телефоне) вспыхивала синим прямоугольником при каждом тапе. У кнопки уже были
стиль наведения, кольцо фокуса и трансформация на `:active`, и вспышка ложилась
поверх всего этого и сводила работу на нет. Элемент выглядел собранным вручную
ровно до того момента, как вы к нему прикасались, — тут платформа отпечатывала
поверх свой шаблон.

`-webkit-tap-highlight-color` — это свойство, которое его рисует. Оно
наследуется, и именно это позволяет исправить проблему чисто. Задайте его
один раз на корне, и каждый потомок унаследует значение, включая содержимое
кастомных элементов в shadow DOM, поскольку наследуемые свойства пересекают
границу. Одна строка убирает вспышку повсюду — не нужно гоняться за ней по
каждому элементу управления.

## Как применить

Задайте прозрачное значение на корне, в вашем глобальном reset:

```css
html {
  -webkit-tap-highlight-color: transparent;
}
```

Это и есть всё исправление вспышки. Теперь верните обратную связь, которую вы
только что убрали, потому что касание всё равно должно *ощущаться* как
сработавшее:

```css
.toggle:active {
  transform: scale(0.97);
}
.chip:active {
  border-color: var(--border-strong);
}
```

`:active` действует на всё время касания (и нажатия мышью), так что это
правильная зацепка для обратной связи на нажатие. Держите её дешёвой —
трансформация или сдвиг цвета, — чтобы она никогда не запускала layout.

Кастомные элементы получают значение по наследованию. Если вы хотите, чтобы
компонент стоял на своих ногах и оставался корректным, когда его уронят на
страницу, забывшую про reset, повторите объявление на хосте:

```css
:host {
  -webkit-tap-highlight-color: transparent;
}
```

## Антипаттерны

```css
/* Removing the focus ring along with the flash. This breaks keyboard and
   switch users — they lose all indication of where focus is. The flash is a
   touch artefact; :focus-visible is an accessibility requirement. Keep it. */
* {
  -webkit-tap-highlight-color: transparent;
  outline: none; /* never */
}
```

```css
/* Setting an opaque tap-highlight to "theme" it. You cannot match your radius
   or padding, it still clips to the box, and it is inconsistent across engines.
   Transparent + your own :active is the only reliable result. */
a {
  -webkit-tap-highlight-color: rgba(0, 0, 0, 0.1);
}
```

```css
/* Repeating the declaration on dozens of selectors because you forgot it
   inherits. One rule on html is enough; the rest is noise. */
a,
button,
.card,
.chip,
.icon-button {
  -webkit-tap-highlight-color: transparent;
}
```

## Как удержать

Поместите объявление в единственный глобальный reset, а затем проверьте две
регрессии. Контур `:focus-visible` должен сохраниться, чтобы клавиатурный фокус
оставался видимым, и каждый интерактивный элемент управления должен сохранить
состояние `:active` (или эквивалент), чтобы тактильная обратная связь не
пропала молча. Быстрый ручной проход на живом устройстве, с тапом по каждому
типу элемента, ловит обе быстрее любого правила линтера.
