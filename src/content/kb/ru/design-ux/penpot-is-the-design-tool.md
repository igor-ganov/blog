Penpot (penpot.app) — это open-source инструмент дизайна, self-hosted альтернатива
Figma. Команда держит локальный, self-hosted экземпляр Penpot, и именно там идёт
работа над дизайном: создаются компоненты, связываются прототипы, поддерживаются
токены дизайна. Работа стекается в Penpot, а не обтекает его.

На проекте стадии дизайна (2026-04-26) инструмент называли «пинпод» и
«penpod». И то, и другое — это Penpot, протранслитерированный через русскую фонетику.
Название не распознали, поэтому в ответ предложили рабочее пространство на Angular и
Storybook. Промах вышел двойным. Инструмент опознали неправильно, да и фаза была не та
(см. [Фаза дизайна — это не фаза кода](/kb/design-ux/design-phase-is-not-code-phase)).
Правильным шагом было спросить URL локального Penpot и продолжить работу внутри
инструмента дизайна.

## Почему это важно

### Что умеет Penpot для дизайна

Penpot поддерживает:
- **Компоненты** — главные компоненты с вариантами, управляемые через булевы пропсы,
  пропсы переопределения текста и подмену вложенных компонентов. Это близко повторяет
  систему компонентов Figma.
- **Токены дизайна** — файлы токенов в формате W3C Design Tokens Community Group
  (DTCG); их можно импортировать и экспортировать как JSON.
- **Прототипирование** — связи между фреймами с типами переходов (мгновенный, растворение,
  слайд), задержкой, оверлеями со скроллом, фиксированным позиционированием.
- **Auto-layout** — раскладка на основе flex внутри фреймов, аналог auto-layout в Figma.
- **Ассеты** — экспорт SVG из любой фигуры; общие библиотеки между страницами и файлами.
- **Формат файла** — `.penpot`-файлы это ZIP-архивы с EDN-данными и ассетами;
  импортируются через диалог импорта Penpot.

Зная эти возможности, понимаешь, какие артефакты подходят. У запроса «добавь токен
цвета» есть нативный для Penpot ответ (добавить его в набор токенов на панели ассетов
или экспортировать изменённый JSON токенов), а не Angular-ответ.

### Варианты написания

Транслитерация «Penpot» в команде гуляет. Всё перечисленное ниже означает Penpot:

- пинпод / penpod / pinpot / пенпот / penpot

Когда что-то из этого всплывает, не считайте это незнакомым инструментом. Читайте как
Penpot и продолжайте. Если локальный URL ещё неизвестен, спросите его.

## Как применять

Есть четыре способа работать с локальным экземпляром Penpot.

### 1. Сгенерировать JSON W3C-токенов дизайна

Для работы на уровне токенов (палитры цветов, типографические шкалы, шкалы отступов,
наборы теней) отдавайте JSON-файл в формате W3C DTCG. Панель токенов Penpot
импортирует его напрямую.

```jsonc
// tokens.json — W3C DTCG format, importable into Penpot
{
  "color": {
    "brand": {
      "ink":    { "$value": "#1a2236", "$type": "color" },
      "accent": { "$value": "#f59e0b", "$type": "color" },
      "surface":{ "$value": "#f9fafb", "$type": "color" }
    },
    "neutral": {
      "100": { "$value": "#f3f4f6", "$type": "color" },
      "300": { "$value": "#d1d5db", "$type": "color" },
      "500": { "$value": "#6b7280", "$type": "color" },
      "700": { "$value": "#374151", "$type": "color" }
    }
  },
  "spacing": {
    "xs":  { "$value": "4px",  "$type": "dimension" },
    "sm":  { "$value": "8px",  "$type": "dimension" },
    "md":  { "$value": "16px", "$type": "dimension" },
    "lg":  { "$value": "24px", "$type": "dimension" },
    "xl":  { "$value": "32px", "$type": "dimension" },
    "2xl": { "$value": "48px", "$type": "dimension" },
    "3xl": { "$value": "64px", "$type": "dimension" }
  }
}
```

Чтобы импортировать: панель ассетов Penpot → Tokens → Import token set → выбрать
JSON-файл.

### 2. Сгенерировать SVG-ассеты

Для иконок, иллюстраций или схематичных диаграмм отдавайте чистые SVG-файлы. Penpot
импортирует SVG как векторные объекты, сохраняя пути, группы и заливку/обводку. SVG
для инструмента дизайна должны:
- Использовать `currentColor` для заливок, которые должны наследовать цвет компонента.
- Обходиться без встроенных растровых изображений (только чистые пути).
- Держать viewBox чистым (без неявных трансформаций от артборда экспортирующего инструмента).
- Использовать единую толщину обводки (например, `stroke-width="1.5"`) по всему набору иконок.

```svg
<!-- icon-home.svg — clean, importable into Penpot as a vector component -->
<svg xmlns="http://www.w3.org/2000/svg"
     width="24" height="24" viewBox="0 0 24 24"
     fill="none" stroke="currentColor"
     stroke-width="1.5" stroke-linecap="round" stroke-linejoin="round">
  <path d="M3 10.5L12 3l9 7.5V21a.75.75 0 0 1-.75.75H15V15H9v6.75H3.75A.75.75 0 0 1 3 21V10.5Z"/>
</svg>
```

### 3. Управлять Penpot через браузерный MCP

Чтобы манипулировать холстом Penpot напрямую (создавать фреймы, размещать компоненты,
править раскладку), управляйте запущенным экземпляром Penpot через браузерный MCP — либо
Playwright, либо Chrome DevTools Protocol. Шаги:

1. Спросите локальный URL Penpot (обычно `http://localhost:7070` или настроенный
   порт) и учётные данные, если они ещё не даны.
2. Браузерным MCP перейдите к файлу и откройте холст дизайна.
3. Взаимодействуйте через веб-интерфейс Penpot: выбирайте инструменты на тулбаре, создавайте
   фигуры, задавайте значения заливки на панели дизайна, связывайте переходы прототипа.

Веб-интерфейс Penpot — это канонический интерфейс, и управление им через MCP ничем не
отличается от работы человека. Не правьте `.penpot`-ZIP-файлы напрямую, пока Penpot держит
файл открытым; при следующем сохранении Penpot затрёт ваши изменения.

### 4. Сгенерировать .penpot-файлы для импорта

Для крупных артефактов вроде полной библиотеки компонентов или набора страниц можно
сгенерировать `.penpot`-файл с нуля. Формат — это ZIP-архив:

```
file.penpot
├── manifest.json   # file metadata, page list, component registry
├── data/
│   └── <file-uuid>.edn  # EDN-serialised design tree
└── media/
    └── <uuid>.<ext>     # embedded raster assets
```

Структуру EDN тяжело писать руками, поэтому такой подход окупается только тогда, когда
вы стартуете с готового `.penpot`-шаблона или можете собрать структуру скриптом. Для
большинства задач JSON W3C-токенов плюс SVG-ассеты покрывают то, что нужно отдать, вообще
без знания формата файла.

### Как спросить URL и учётные данные

Если локального URL Penpot нет в контексте проекта, а задача требует работы с запущенным
экземпляром, спросите ровно так:

> «Какой локальный URL у вашего экземпляра Penpot и есть ли у вас учётные данные, которые
> я могу использовать?»

Не предполагайте `localhost:7070` без подтверждения. Порт настраивается, и экземпляр
может стоять за обратным прокси.

## Антипаттерны

**Не распознать инструмент**

Увидеть «пинпод» или «penpod», принять это за незнакомый инструмент и предложить другой
тулчейн — это та самая ошибка, описанная выше. Опознайте инструмент, потом работайте с ним.

**Предлагать установить другой инструмент дизайна**

Если пользователь работает в Penpot, не предлагайте Figma, Sketch или Adobe XD как
альтернативы. Инструмент уже выбран, поэтому задача — работать внутри него.

**Генерировать Storybook-истории или каркасы компонентов для задач дизайна**

Запрос «создай компонент карточки в Penpot» — это задача дизайна. Правильный артефакт —
нативная для Penpot спецификация компонента (оси вариантов, настройки auto-layout, ссылки
на токены). Файл `.stories.ts` — не артефакт дизайна.

**Отдавать CSS как артефакт дизайна**

CSS — это артефакт реализации, а не дизайна. Передать блок CSS-кастомных свойств в задачу
дизайна Penpot — значит схлопнуть фазу дизайна в фазу реализации. Артефакт дизайна — это
JSON токенов, а CSS, который их потребляет, появляется позже, отдельным документом для
отдельной фазы.
